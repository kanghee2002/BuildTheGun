## 기술 설계서 (Tech Design)

### 1. 개요
- 이 문서는 **게임의 기술 구조와 아키텍처**를 정의합니다.
- 씬 구조, 계층 구조, 데이터 흐름, 성능/툴링 방식을 정리합니다.

### 2. 프로젝트 구조
- **폴더 구조(예시)**:
  - `Scripts/Core`: 공통 코어 로직(시간, 이벤트, 유틸, DI 등)
  - `Scripts/Game`: 도메인 로직(전투, 무기, 인벤토리 등)
  - `Scripts/Presentation`: UI, 애니메이션, 이펙트 트리거
  - `Scripts/Infrastructure`: 저장, 네트워크, AI 연동, 분석 툴
  - `ScriptableObjects`: 밸런스/정적 데이터
- 실제 구조를 여기에 확정하고, 변경 시 히스토리를 남깁니다.

### 3. 씬 & 게임 상태 구조
- **씬 구성**:
  - 예: `Boot`, `MainMenu`, `Gameplay`, `Result`
- **게임 상태 관리 방식**:
  - 예: GameState(메뉴/로딩/플레이/일시정지/결과) enum + 상태머신
- 씬 간 전환 규칙, 싱글턴/서비스 로케이터 사용 여부 등 정의

### 4. 계층 구조 & 의존성 규칙
- **계층**:
  - 도메인(Game) → 프레젠테이션(Presentation) → Unity 컴포넌트
- **의존성 규칙**:
  - 도메인 로직은 Unity API에 직접 의존하지 않는다.
  - Presentation 레이어에서만 MonoBehaviour, GameObject, Transform 등을 사용.
  - Infrastructure는 도메인 인터페이스에 의존하고, 구현체를 제공한다.

### 5. 데이터 & 저장 구조
- **정적 데이터**:
  - ScriptableObject/JSON/기타 방식
  - 예: 무기 정의, 적 정의, 맵 설정
- **런타임 데이터(도메인 모델)**:
  - PlayerState, WeaponState 등
- **세이브/로드 방식**:
  - 저장 포맷(JSON/Binary 등), 저장 위치, 버전 관리 전략

### 6. AI/외부 서비스 연동(선택)
- **AI 적용 지점**(예: 밸런스 테스트용 시뮬레이션, 대사 생성, 빌드 퀄리티 체크 등)
- **외부 서비스**:
  - 분석(Analytics), 클라우드 저장, 매치메이킹 등

### 7. 성능 & 품질 기준
- **프레임 타겟**:
- **메모리 예산**:
- **GC/할당 정책**:
  - Update 내 new 사용 금지, 풀링 전략 등

### 8. 로깅/디버깅/툴링
- **로깅 규칙**:
  - 로그 레벨, 태그, 파일/네트워크 로그 저장 여부
- **디버그 툴**:
  - 인게임 디버그 메뉴, 개발용 치트, 프로파일링 방식
