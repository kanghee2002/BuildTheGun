## 아이디어 메모 (자유롭게 적기)

### 사용 규칙
- 이 문서는 **떠오르는 아이디어를 검열 없이 빠르게 적는 곳**입니다.
- 문장/형식이 거칠어도 괜찮습니다. 대신, 나중에 실행 가능하도록 최소한의 맥락만 남깁니다.
  - 기대 효과(왜 좋은지)
  - 적용 화면/상황(언제 보이는지)
  - 구현 난이도/리스크(알면)

---

### UI/UX 아이디어

#### 합체 카드 비주얼/인터랙션
- 합체 카드는 카드 테두리가 겹쳐 보이는 **“레이어드(Layered)” 비주얼**로 표시합니다.
  - 2장 합체: 2겹
  - 3장 합체: 3겹
- **빌드 편집 화면 / 일시정지 화면**처럼 플레이어가 꼼꼼히 볼 수 있는 순간에는,
  - 합체 카드를 **탭(모바일 대응) 또는 호버(PC)** 하면,
  - 개별 구성 카드가 **펼쳐져 보이도록** 표시합니다.

### AI/툴링 & 공고 어필 포인트 (자연스러운 누적)
#### 목표
- “AI를 구현”하기보다, 개발 과정에서 AI가 쓸모 있는 형태(검증/실험/리포트)에 붙도록 파이프라인을 차근차근 쌓기
- 데이터 기반 밸런싱/QA/분석 자동화를 포트폴리오/이력서에서 설명 가능하게 만들기

#### 1단계: 데이터 스키마를 검증 가능하게 만들기 (시뮬레이터 없어도 됨)
- 카드/총기/유물 효과를 텍스트가 아니라 스펙 필드로 고정
  - 예: `type`, `trigger`(OnHit/OnFire/OnReload 등), `keywords`, `params`, `constraints`(상한/금지 조합)
- 최소 “검증기(Validator)”부터 만들기
  - 누락 필드 탐지
  - 키워드 타입/트리거 호환성 체크
  - 밸런스 리스크(무한 루프, 과도한 CC, 상한 없는 스택 등) 후보 탐지
- Cursor 사용 흐름(정직하게 표현)
  - 스키마/검증 로직의 초안을 빠르게 만들고, 결과를 내 코드/문서로 남기기

#### 2단계: 테스트용 전투 러너(또는 밸런스 실험 러너) 만들기 (없다면 개발 예정)
- 고정된 빌드/덱/시나리오를 N번 반복 실행
- 러너 출력(기계가 읽는 형태 + 사람이 보는 리포트 둘 다)
  - DPS/TTK(또는 처치 시간), 재장전 횟수, 상태이상 누적 등
  - 전투/이벤트 로그(분기 조건/트리거 발생 시점 포함)
- “재현 가능한 테스트/측정 도구화”가 핵심 어필 포인트

#### 3단계: 리포트 생성(AI 활용을 가장 공고 친화적으로 연결)
- 러너가 만든 지표/로그를 입력으로 “이상 조합 원인 후보 + 다음 실험 제안”을 리포트로 자동 생성
- 산출물은 Markdown 파일로 저장해서 기록 남기기(예: `Reports/balance_run_YYYY-MM-DD.md`)

#### 데이터 포맷 선택 가이드(어필 관점)
- 추천: “하이브리드”
  - 소스 오브 트루스: `JSON(또는 YAML/CSV)`로 두고 스키마/검증/시뮬레이션 입력에 최적화
  - 런타임/에디터 편의: Unity에선 `ScriptableObject`로 변환(필요 시)
- 이유: 공고에서 좋아하는 “AI가 바로 먹을 수 있는 데이터 + 검증/실험 파이프라인”을 만들기 쉬움
